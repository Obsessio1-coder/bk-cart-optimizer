# Настройки Burger King Optimizer

## Тестирование
```powershell
chcp 65001 >$null; python -X utf8 optimizer.py Воппер Кола Наггетсы
```

## Архитектура

### Источник данных
- Загрузка: `bk_api.load_data(rid, mode)` — сначала live API (если mode=live/auto), затем кэш `bk_all_menus/menu_{rid}.json`.
- Цена комбо: `menu.result.combos[].main_info.price` (копейки → рубли делением на 100).
- Состав слотов (какие блюда входят в комбо) берётся из `combo_structures.json` по `combo.id`.
- MonoCoupons читаются из `general_coupons[]` ресторана; структура — из `struct['coupons']` по ключу `code`.

### Ценообразование
- **ComboCoupon**: `combo_price` (из меню ресторана) + `UPSELL` (из статической матрицы, если заменён гарнир) + `sauce_cost` (если есть соусный слот).
- **MonoCoupon**: цена купона из `general_coupons[]` (если `dish_id` купона есть в меню ресторана).
- Никаких калибровок, наценок, вычислений base+markup — только прямые цены ресторана.

### UPSELL_MATRIX (хардкод)
```python
UPSELL_MATRIX = {
    "small_fries":  {"nuggets_3": 4.99, "nuggets_6": 49.99, "nuggets_9": 79.99, "medium_fries": 29.99, "large_fries": 49.99, "maxx_fries": 119.99},
    "medium_fries": {"nuggets_3": 0.0,  "nuggets_6": 19.99, "nuggets_9": 39.99, "large_fries": 39.99, "maxx_fries": 89.99},
    "large_fries":  {"nuggets_6": 0.0,  "nuggets_9": 19.99, "maxx_fries": 49.99},
}
```
- Теги определяются функцией `tag_side(name)` по ключевым словам в названии блюда.
- Если замены нет в матрице — `DEFAULT_UPCHARGE = 19.99`.

### Инициализация (строгая фильтрация)
1. **build_menu_index** — загрузка меню ресторана. Позиции с ценой 0 (out of stock) полностью исключаются.
2. **build_combo_prices** — собирает цены всех комбо и купонов из меню ресторана.
3. Для каждого комбо из `menu.combos`:
   - Ищется структура в `struct['combos']` по `id`.
   - Фильтрация слотов: блюда не из меню ресторана или с ценой 0 отсеиваются. Пустой слот → комбо отбрасывается.
   - Соусные слоты: если 100% блюд в слоте — соусы (`соус`/`кетчуп` в названии) и их ≥2, слот маркируется как `sauce_slot_indices`.
   - Дефолтный наполнитель: для гарнира — cheapest/standard potato; для напитка — cheapest cola/soda.
   - Цена комбо — **только** из меню ресторана.

### Оптимизация (2 этапа)
1. **MonoCoupon как price floor**: для каждого заказанного блюда проверяется `discount_map[dish_id]` (купон из `general_coupons` + `struct['coupons']`). Если есть — эффективная цена блюда снижается до купонной.
2. **ComboCoupon search**: комбинаторный перебор (до 3 комбо). Для каждого совместимого набора:
   - Берётся `combo_price` из меню ресторана.
   - Если гарнир заменён на не-дефолтный: добавляется `UPSELL_MATRIX[default_tag][chosen_tag]`.
   - Если есть соусный слот: считается 2 сценария (с соусом / без).
   - К остатку применяются MonoCoupon.
3. Выводится лучший вариант (минимальная итоговая стоимость).

### Матчинг
- **Точный** (по названию) — наивысший приоритет
- **Quantity-aware**: "Наггетсы (6 шт.)" → "Наггетсы (12 шт.)" (больше — ок)
- **Fuzzy word-subset**: "Воппер" → "Воппер с сыром" (апгрейд), "Кола малый" → "Кола мал" (стемминг)
- **Запрет cross-type**: "Воппер с сыром" НЕ матчится с "Воппер Джуниор" (разные уникальные слова)

## Команды
### CLI
```powershell
chcp 65001 >$null; python -X utf8 optimizer.py Воппер Кола Наггетсы
python -X utf8 optimizer.py "Кола 0,4" "Наггетсы 9 шт"
python -X utf8 optimizer.py Рожок
```

### API (локально)
```powershell
python -m flask --app api/index.py run --port 5000
# Оптимизация
curl -X POST http://127.0.0.1:5000/optimize ^
  -H "Content-Type: application/json" ^
  -d "{\"items\":[\"Воппер\",\"Кола\",\"Наггетсы\"]}"

```

## Известные ограничения
- MonoCoupon привязывается строго по dish_id. Если блюда нет в меню ресторана — купон не применяется
- Coupon в struct['coupons'] ключится по `code`, а combo в struct['combos'] — по `id` (разные системы)
- UPSELL_MATRIX — статическая; не учитывает региональные различия цен на замену гарнира
- Матчинг не различает «Воппер» и «Воппер с сыром» (считается апгрейдом, показывается стрелкой `→`)
- Количество наггетсов без указания → выбирается наименьшая порция из меню (6 шт.)
- Для точного указания использовать кавычки: `"Кола 0,4"` или `"Наггетсы 9 шт"`
